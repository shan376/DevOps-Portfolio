## 1. What is Jenkins? (Introduction)

Jenkins is an open-source automation server used to automate the process of building, testing, and deploying software. It is a core tool in the CI/CD (Continuous Integration/Continuous Deployment) pipeline in DevOps.

### ğŸ”¹ Key Features:

* Automates repetitive DevOps tasks.
* Supports hundreds of plugins (Git, Docker, Kubernetes, etc.).
* Web-based GUI.
* Easy integration with source control (e.g., GitHub).
* Build triggers (via webhooks, schedule, or manually).

### ğŸ“Œ Jenkins in DevOps:

* **CI (Continuous Integration):** Developers push code frequently to a shared repo. Jenkins automatically builds and tests the code.
* **CD (Continuous Deployment):** After testing, Jenkins can deploy the code to staging/production automatically.

---

## 2. Jenkins Architecture

Jenkins follows a **Master-Agent (or Controller-Agent)** architecture.

### ğŸ”§ Components:

* **Master/Controller:** Main server that handles:

  * Scheduling builds
  * Monitoring agents
  * Reporting results
* **Agent/Node:** Executes build jobs sent by the master. Can be:

  * On the same machine (built-in)
  * On another machine (remote agents)

## 3. Trigger Jenkins Jobs Automatically

There are 3 types of triggers:

ğŸ”¹ **1. Manual Trigger:** Click *Build Now* in Jenkins.
ğŸ”¹ **2. Poll SCM (Git polling):** Jenkins checks GitHub on a schedule.

```bash

```

ğŸ”¹ **3. Webhook Trigger (Recommended):** Automatic builds on every push.

### âœ… Setup GitHub Webhook:

1. Go to GitHub repo â†’ *Settings â†’ Webhooks*
