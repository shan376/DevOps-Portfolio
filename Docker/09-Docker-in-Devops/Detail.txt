## **Docker Basics (Part 1)**

### **1. Containers, Images, and Registries in Docker**

**What is Docker?**
Docker is a platform used to develop, ship, and run applications in containers. Containers are lightweight and portable, allowing applications to run consistently across different environments.

**Docker Containers**
A container is a lightweight, standalone package that includes everything needed to run software ‚Äî code, runtime, tools, libraries, and settings.

**Benefits:**

* Portability (run anywhere)
* Consistency across environments
* Isolation between containers

**Commands:**

```
docker run <image_name>
docker ps
docker stop <container_id>
docker rm <container_id>
```

**Docker Images**
Images are snapshots used to create containers. They include applications and dependencies.

**Benefits:**

* Versioning
* Reuse
* Layered architecture

**Commands:**

```
docker build -t <image_name> <path>
docker images
docker rmi <image_id>
```

**Docker Registries**
Registries store and distribute Docker images (like Docker Hub or private registries).

**Commands:**

```
docker login
docker push <image_name>
docker pull <image_name>
```

### **üîß Practical Work: Install Docker and Run a Container**

## **Docker Advanced (Part 2)**

### **1. Dockerfile, Volumes, and Networking**

**Dockerfile**
A script that defines how to build Docker images.
**Benefits:**

* Reproducibility
* Automation
* Version control

**Example Dockerfile:**

```dockerfile
FROM python:3.8-slim
WORKDIR /app
COPY . /app
RUN pip install --no-cache-dir -r requirements.txt
EXPOSE 80
CMD ["python", "app.py"]
```

**Commands:**

* FROM ‚Üí Base image
* COPY ‚Üí Copy files
* RUN ‚Üí Execute commands
* EXPOSE ‚Üí Open port
* CMD ‚Üí Start command

---

**Docker Volumes**
Used to persist data outside of containers.

**Commands:**

```
docker volume create <volume_name>
docker run -v <volume_name>:/path/in/container <image_name>
docker volume ls
docker volume rm <volume_name>
```

---

**Docker Networking**
Enables communication between containers.

**Types:**

* Bridge
* Host
* Overlay

**Commands:**

```
docker network create <network_name>
docker network ls
docker run --network <network_name> <image_name>
```

---

## **Docker Compose (Part 3)**

**What is Docker Compose?**
A tool for defining and managing multi-container applications with a YAML file (`docker-compose.yml`).

**Benefits:**

* Simplifies multi-container setup
* Manages service dependencies
* Ensures consistent environments

---

**Example docker-compose.yml:**

```yaml
version: '3'
services:
  web:
    image: nginx
    ports:
      - "8080:80"
  db:
    image: mysql
    environment:
      MYSQL_ROOT_PASSWORD: example
volumes:
  db-data:
```

---
