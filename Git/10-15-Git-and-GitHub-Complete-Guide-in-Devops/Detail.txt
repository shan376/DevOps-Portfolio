### ğŸ“Œ Topic 1: What is Source Code Management (SCM)?

**Definition:**
Source Code Management (SCM) is the process of tracking, managing, and controlling changes to source code over time.

**Why SCM is Important in DevOps:**

* Keeps a history of code changes
* Helps teams collaborate on the same codebase
* Allows rolling back to previous versions
* Supports automation in CI/CD pipelines

**Real-world Scenario:**
In a team, SCM ensures developers' work is synchronized, traceable, and conflict-free.

### ğŸ“Œ Topic 2: CVCS vs. DVCS

| Feature             | CVCS (Centralized)    | DVCS (Distributed)            |
| ------------------- | --------------------- | ----------------------------- |
| Server              | Single central server | Every developer has full copy |
| Example Tools       | SVN, Perforce         | Git, Mercurial                |
| Internet Required?  | Always                | Only when pushing/pulling     |
| If Server Fails     | Risk of data loss     | No problem                    |
| Collaboration Speed | Slower                | Faster                        |

**ğŸ’¡ Summary:**

* CVCS â†’ One central place = risk if server fails.
* DVCS â†’ Each user has full copy = safe, fast, ideal for DevOps.

### ğŸ“Œ Topic 3: What is Git?

**Definition:**
Git is a Distributed Version Control System (DVCS) used to manage source code.

**Key Features:**

* Tracks changes and history
* Branching and merging
* Collaboration without conflicts
* Works offline and is fast

**Used With:** GitHub, GitLab, Bitbucket

**Example:**
Each developer creates a branch, works independently, then merges into the main branch â€” Git tracks all changes.
